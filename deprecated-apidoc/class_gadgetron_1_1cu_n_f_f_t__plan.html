Detailed Description

template<class REAL, unsigned int D, bool ATOMICS = false>
class Gadgetron::cuNFFT_plan< REAL, D, ATOMICS >

Cuda implementation of the non-Cartesian FFT.

— NFFT class declaration —

REAL: desired precision : float or double D: dimensionality : { 1,2,3,4 } ATOMICS: use atomic device memory transactions : { true, false }

For the tested hardware the implementation using atomic operations is slower as its non-atomic counterpart. However, using atomic operations has the advantage of not requiring any pre-processing. As the preprocessing step can be quite costly in terms of memory usage, the atomic mode can be necessary for very large images or for 3D/4D volumes. Notice: currently no devices support atomics operations in double precision.

Parameters
matrix_sizethe matrix size to use for the NFFT. Define as a multiple of 32.
matrix_size_osintermediate oversampled matrix size. Define as a multiple of 32. The ratio between matrix_size_os and matrix_size define the oversampling ratio for the NFFT implementation. Use an oversampling ratio between 1 and 2. The higher ratio the better quality results, however at the cost of increased execution times.
Wthe concolution window size used in the NFFT implementation. The larger W the better quality at the cost of increased runtime.
devicethe device (GPU id) to use for the NFFT computation. The default value of -1 indicates that the currently active device is used.
