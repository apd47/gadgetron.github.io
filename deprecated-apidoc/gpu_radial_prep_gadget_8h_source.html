19 /*
20  ------------------------------------------
21  Trajectory modes for radial reconstruction
22  ------------------------------------------
23 
24  Mode 0 and Mode 1 are variants of 'fixed' radial trajectories with interframe rotation.
25  Mode 2 and Mode 3 denote radial trajectories with golden ratio based angular profile spacings.
26 
27  Let
28  'i' denote the number of profiles per (undersampled) frame
29  'j' denote the number of frames per trajectory rotation (to obtain a fully sampled acquisition)
30  'h' denote a variable of type ISMRMRD::AcquisitionHeader
31 
32  It is possible to explicitly set 'i' and 'j' in the Gadgetron configuration file.
33  For some modes this is (partly) required,
34  for others they will be automatically determined from the incoming profile headers.
35 
36  Mode 0:
37  -------
38  For each rotation cycle profiles are numbered using the scheme
39 
40  0+0*j,0+1*j,0+2*j,...,0+(i-1)*j, (1st frame)
41  1+0*j,1+1*j,1+2*j,...,1+(i-1)*j, (2nd frame)
42  2+0*j,2+1*j,2+2*j,...,2+(i-1)*j, (3rd frame)
43  ...,
44  (j-1)+0*j,(j-1)+1*j,(j-1)+2*j,...,(j-1)+(i-1)*j
45 
46  as given in h.idx.kspace_encode_step_1.
47  Both 'i' and 'j' are automatically derived and thus need not be explicitly specified in a configuration file.
48  For mode 0 both 'i' and 'j' can be changed dynamically as desired e.g. for real-time imaging.
49 
50  Mode 1:
51  -------
52  Profiles are numbered 0,1,2,...,i-1, 0,1,2,...,i-1, ... as given in h.idx.kspace_encode_step_1.
53  'j' is estimated as 'matrix_size'/'i' and should be explicitly set in the configuration file if this is not the case, e.g.:
54  <property><name>frames_per_rotation</name><value>8</value></property>
55 
56 
57  Mode 2 and Mode 3:
58  -------
59  Profiles are numbered
60  0,1,2,...,i-1, 0,1,2,...,i-1, 0,1,2,...,i-1, ...
61  or
62  0,1,2,...,i-1, i,i+1,i+2,...,2*i-1, 2*i,2*i+1,2*i+2,3*i-1, ...
63  as given in h.idx.kspace_encode_step_1.
64  'i' should be explicitly specified in the Gadgetron configuration file, e.g.:
65  <property><name>profiles_per_frame</name><value>32</value></property>
66  If not it defaults to i=32.
67  'j' is explicitly set to '1' even if specified in the configuration file.
68 */
